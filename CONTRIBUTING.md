# Contributing to iLINKZIP

## Setup

1. `npm install`
2. `cp .env.example .env`
3. `npm run dev`

## Before opening a PR

1. `npm run check`
2. `npm test`
3. `npm run build`

## Pull Request Rules

- Keep PR scope focused.
- Add or update tests for behavior changes.
- Update `README.md` when API or deployment behavior changes.
- Do not commit secrets or `.env` files.
